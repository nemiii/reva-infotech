Below is a practical website brief you can use to build your marketing site, using the visual direction of Specify
  ’s landing page as the reference theme: https://specifyapp.com/?ref=landingfolio

  What I’m taking from that reference:

  - premium SaaS presentation
  - clean, high-trust enterprise tone
  - strong hero section
  - layered cards and soft shadows
  - product-led storytelling
  - customer logos, proof, testimonials, case-study sections
  - polished gradients, rounded surfaces, modern typography
  - clear CTA flow from awareness to demo/contact

  1. Website Goal
  Build a marketing website for the software platform that:

  - explains the product clearly
  - presents all major modules, services, and business value
  - positions the product as enterprise-grade
  - drives demo requests, contact inquiries, and sales conversations

  2. Suggested Site Structure

  1. Home
  2. Platform Overview
  3. Features / Modules
  4. Industries / Use Cases
  5. Services
  6. Integrations
  7. Security / Compliance
  8. Pricing or “Talk to Sales”
  9. Case Studies / Testimonials
  10. About
  11. Contact / Demo Request

  3. Content Framework
  For each module/service in your application, capture:

  - module name
  - business problem solved
  - key workflows
  - main roles using it
  - measurable outcomes
  - integrations/dependencies
  - screenshots or UI previews

  Example feature buckets:

  - Procurement
  - Inventory
  - Manufacturing
  - Maintenance
  - Mobility / Fleet
  - Finance
  - Workforce
  - Admin / Workflow / Master Data
  - Reports / Analytics
  - Multi-site operations

  4. Design Direction
  Use the Specify theme as inspiration, not as a copy.

  Design rules:

  - premium B2B SaaS
  - modern, minimal, elegant
  - restrained gradients
  - clean white or soft off-white sections
  - dark accent areas for hero / major CTA
  - rounded cards, polished spacing, high readability
  - product screenshots as the core visual proof
  - avoid clutter and dense text walls

  5. Prompt To Generate The Website
  Use this prompt in ChatGPT, Claude, Lovable, v0, Framer AI, or your design/dev workflow:

  Create a premium B2B SaaS marketing website for an enterprise business operations software platform.

  Design inspiration:
  Use the visual quality and interaction feel of https://specifyapp.com/?ref=landingfolio as inspiration only. Do not
  copy content or branding. Match the same level of polish, trust, spacing, premium gradients, rounded cards, elegant
  SaaS layout, strong hero, customer-proof sections, and modern enterprise storytelling.

  Website goals:
  - Explain the product clearly
  - Showcase all modules and services
  - Position the platform as enterprise-grade
  - Drive demo bookings and contact requests

  Brand positioning:
  This platform is a unified enterprise operations suite for managing procurement, inventory, warehouses, stock
  movement, manufacturing, production orders, work orders, maintenance, assets, incidents, fleet/mobility, finance,
  workforce, workflow approvals, master data, reporting, and multi-site operations.

  Target audience:
  - CEOs / Founders
  - COOs / Operations leaders
  - Supply chain managers
  - Procurement heads
  - Plant / factory managers
  - Finance controllers
  - Maintenance heads
  - Warehouse managers
  - Enterprise IT / digital transformation teams

  Tone:
  - Enterprise
  - Confident
  - Clear
  - High-trust
  - Outcome-focused
  - Not overly technical
  - Not generic startup language

  Required sections:
  1. Hero
  2. Customer logos / trust strip
  3. Platform overview
  4. Core modules grid
  5. Workflow / automation section
  6. Industry use cases
  7. Services section
  8. Integrations section
  9. Analytics / reporting section
  10. Security / governance section
  11. Testimonials / proof
  12. CTA section
  13. Footer

  Hero requirements:
  - strong enterprise headline
  - short supporting copy
  - primary CTA: Book a Demo
  - secondary CTA: Talk to Sales
  - large product-style visual or dashboard composition
  - premium SaaS look

  Features/modules section:
  Show major modules as polished cards with concise descriptions:
  - Procurement
  - Inventory
  - Warehousing
  - Manufacturing
  - Maintenance
  - Mobility / Fleet
  - Finance
  - Workforce
  - Workflow & Audit
  - Master Data
  - Reporting & Analytics
  - Multi-site Operations

  Services section:
  Include implementation and support services such as:
  - Product implementation
  - Process consulting
  - Data migration
  - ERP/workflow integration
  - Custom module delivery
  - Training and onboarding
  - Ongoing support

  Design system:
  - premium typography
  - soft gradients
  - elegant shadows
  - rounded cards
  - generous spacing
  - polished SaaS components
  - responsive across desktop/tablet/mobile

  Output:
  - full sitemap
  - section-by-section copy draft
  - visual design direction
  - component list
  - CTA strategy
  - recommended image/screenshot usage
  - homepage wireframe
  - page-wise structure for all core pages

  6. Step-Wise Document To Build The Website

  Step 1: Define positioning
  Write:

  - one-line product summary
  - who it is for
  - what business outcomes it improves
  - why it is different

  Output:

  - headline
  - subheadline
  - 3 primary value pillars

  Step 2: Inventory all modules
  Create a spreadsheet with:

  - module
  - purpose
  - top 5 features
  - target user
  - business value
  - screenshot availability

  Step 3: Group modules into website-ready categories
  Example:

  - Supply Chain: procurement, inventory, warehouse
  - Production: BOM, routing, production orders, work orders
  - Asset Reliability: maintenance, incidents, repair logs
  - Enterprise Control: finance, admin, workflow, master data
  - People & Mobility: workforce, fleet, drivers, vehicles
  - Intelligence: reports, analytics, cross-site views

  Step 4: Collect proof
  Gather:

  - customer names/logos
  - metrics
  - implementation wins
  - testimonials
  - screenshots
  - industries served

  Step 5: Write the homepage
  Homepage should answer:

  - what is it
  - who is it for
  - what does it cover
  - why trust it
  - what next action should be taken

  Step 6: Create feature pages
  One page per major area:

  - overview
  - workflows
  - screenshots
  - benefits
  - role-based value
  - CTA

  Step 7: Create services page
  Cover:

  - discovery
  - implementation
  - migration
  - integration
  - customization
  - training
  - managed support

  Step 8: Create use-case / industry pages
  Examples:

  - manufacturing
  - distribution
  - warehouse-heavy businesses
  - multi-site operations
  - maintenance-intensive operations

  Step 9: Add CTA system
  Use consistent CTAs:

  - Book a Demo
  - Talk to Sales
  - Request Consultation
  - Explore Modules

  Step 10: Final visual pass
  Check:

  - consistency
  - product-first visuals
  - trust cues
  - spacing
  - mobile responsiveness
  - page speed
  - form clarity

  7. Homepage Copy Skeleton

  - Hero: Unified operations platform for procurement, inventory, manufacturing, maintenance, finance, workforce, and
    multi-site control
  - Trust strip: customers / industries / deployment confidence
  - Platform section: one connected system across departments
  - Modules grid: all core areas
  - Automation section: approvals, workflows, auditability
  - Analytics section: live reporting and decision visibility
  - Services section: implementation to support
  - Testimonial section
  - Final CTA

  8. Deliverables You Should Create

  - sitemap

  - homepage wireframe
  - full page content draft
  - image/screenshot plan
  - feature taxonomy
  - CTA strategy
  - design system notes

  9. Best Practice
  Do not make it a generic “ERP software” site. Make it:

  - operational
  - specific
  - module-driven
  - proof-based
  - visually premium
